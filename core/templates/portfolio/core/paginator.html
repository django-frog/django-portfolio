{% comment %} Core Pagination Component -------------------------- This component is designed to work **only** with Django's class-based views that use pagination. It dynamically displays page numbers, handles navigation, and retains query parameters. Required context variables: - `page_obj` → The current paginated page (from Django's pagination system). - `paginator` → The paginator instance that manages all pages. - `query_string` → A string containing additional GET parameters (e.g., search filters) to persist across pages. Notes: - If there is no previous or next page, the respective buttons will be disabled. - The current page is highlighted visually. {% endcomment %}
